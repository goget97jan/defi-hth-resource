# defi-hth-resource

## 项目简介

defi-hth-resource 是一个用于归档和发布多个独立 HTML 页面的资源仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的存储与展示空间。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 归档的 HTML 页面存放目录
│   ├── example-page-1.html
│   ├── example-page-2.html
│   └── ...
└── assets/             # 页面依赖的静态资源（CSS、图片、字体等）
    ├── styles/
    ├── images/
    └── ...
```

- `pages/` 目录：存放所有独立 HTML 页面文件，每个页面可自包含样式与脚本。
- `assets/` 目录：供页面共享使用的静态资源，按类型或用途组织子目录。

## 页面归档说明

- 所有归档页面均为独立的 HTML 文件，可直接通过浏览器打开或部署至任意静态托管服务。
- 页面内容应由用户自行准备并提交，本仓库仅提供存储与版本管理功能。
- 页面之间无强制依赖关系，每个页面均可独立运行。

## 维护说明

- 本仓库欢迎通过 Pull Request 提交新的页面归档，或对现有页面进行更新。
- 提交的页面应保持基本的结构完整性与可读性，避免包含恶意代码或违反相关政策的内容。
- 对于不再需要的页面，可通过 Issue 提出移除请求，或由维护者定期清理。

## 部署

本仓库可直接使用 GitHub Pages 或其他静态托管服务进行部署。部署后，通过 `https://<你的用户名>.github.io/defi-hth-resource/` 即可访问仓库根目录下的页面。
